# Verification Rules

이 프로젝트에서 기능 개발, 수정, 리팩토링을 수행한 뒤에는 변경된 앱별로 타입 검사와 lint 검사만 실행한다.

## 일반 작업 후 검증

- 기능 개발, 버그 수정, 리팩토링 후에는 영향을 받은 각 앱의 `typecheck`와 `lint`를 실행한다.
- 루트에서 실행 가능한 짧은 명령이 있으면 우선 사용한다.
  - 예: `pnpm typecheck:web`
  - 예: `pnpm --filter web lint`
- 일반 작업 완료 검증 단계에서는 사용자가 별도로 요청하지 않는 한 `build`를 실행하지 않는다.
- 테스트가 필요한 변경이라고 판단되면 사용자 요청이나 관련 룰에 따라 별도로 테스트 실행 여부를 판단하되, 기본 완료 검증은 타입 검사와 lint 검사로 제한한다.

## GitHub 업로드 전 검증

- 사용자가 깃허브에 올리기, push, PR 생성, 배포 브랜치 반영을 요청한 경우에는 업로드 전에 반드시 빌드 검증을 실행한다.
- 빌드 검증은 영향을 받은 앱 또는 패키지 범위로 실행하되, 범위가 불명확하면 루트의 전체 빌드 명령을 사용한다.
  - 예: `pnpm --filter web build`
  - 예: `pnpm build`
- 빌드가 실패하면 깃허브에 올리지 않고 실패 원인과 필요한 조치를 먼저 보고한다.
