# 하네스

이 폴더는 우리의 의료 노트의 AI/OCR/권한 동작을 반복적으로 검증하기 위한 평가 자산을 모읍니다.

## 목적

- AI가 사용자가 제공하지 않은 의료 사실을 추가하지 않는지 확인합니다.
- OCR로 추출한 약 정보가 원문과 얼마나 일치하는지 확인합니다.
- 기록 작성자, 기록 대상자, 그룹 공유 권한이 섞이지 않는지 확인합니다.
- 3/6/12개월 리포트가 기록 기반 추이만 요약하고 의학적 판단을 하지 않는지 확인합니다.
- 프롬프트, 모델, 스키마 변경 전후의 회귀를 기록합니다.

## 폴더 구조

```text
harness/
  evals/       # 평가 묶음과 케이스 정의
  fixtures/    # 입력 예시와 기대 결과
  prompts/     # AI 작업별 프롬프트 초안
  runs/        # 하네스 실행 기록
  scripts/     # 실행 도구가 들어갈 자리
```

## 기본 실행 흐름

1. `docs/HARNESS.md`에서 평가 원칙을 확인합니다.
2. 작업할 기능에 맞는 `harness/evals/*/README.md`를 읽습니다.
3. 관련 픽스처를 `harness/fixtures/`에서 확인합니다.
4. 프롬프트를 바꿨다면 `harness/prompts/`에 변경 이유를 남깁니다.
5. 실행 결과는 `harness/runs/YYYY-MM-DD-name/`에 기록합니다.

## 릴리스 차단 후보

- 선택하지 않은 그룹에 의료 노트가 노출됩니다.
- 기록 대상자가 포함되지 않은 그룹에서 해당 대상자의 기록이 보입니다.
- AI가 진단명이나 치료 지시를 새로 만들어냅니다.
- OCR 약 정보가 확정값처럼 저장되며 사용자 검토 흐름이 없습니다.
- 장기 리포트가 기록 기반 추이를 넘어 의학적 판단을 합니다.
